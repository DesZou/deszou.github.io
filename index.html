2.1.1 7日

今天看到一个比较有意思的 c++ 库 fc++ ，用 c++ 来实现函数式编程。 毫无疑问是用到了 template ，这点是跑不了的，多态什么的都得靠它。 但是让我吃惊的是它不需要用到任何新的 c++ 特性， 官网上的测试编译器只用到了 g++3 ！ （也说明了这个东西已经不新了） 不过有点坑的是它没有 任何 标准文档，就算是英文的也比没有要强得多啊。 但是它除了库文件之外还有一个文件夹的例子，里面有很多特性的示例， 也许读代码比文档会更好吧。

2.1.11 28日

连写两天 io 。今天竟然考了我的题目，真是“受宠若惊”了，呵呵。 托考试的福，今天上午总算撸了点正经玩意：可持久化 Treap 。 虽然很水（那又怎么样呢），实现的还比较满意。缺点就是没有用指针， 但是我好不容易有一个这么好写的数据结构，实在不忍心让它变复杂。 可持久化的部分很好加入，只要把 split 里面的返回值变成原节点的 copy 就行了。

今天还在 cppreference 里淘了会儿宝，发现了两个好东西：第一个是 std::make_heap ， std::push_heap ， std::pop_heap 这一组函数，可以用自己定义的空间维护一个堆。 速度非常不错（至少和 std::priority_queue 比起来），以后再也不用手写堆了。 PS ：好像 std::priority_queue 存在的原因是因为上面那组函数可能会被强制破坏堆的性质。 可是它用了 std::vector 管理内存，这就让它有点鸡肋了（慢是原罪）。 第二个是 std::valarray ，其实用处不会很大。但是让我知道 c++ 有这个东西心里会舒服一点。 它本来是设计来进行数学运算的，定义了各种运算符号。让我觉得舒服的是 std::slice ！ 切片！ python 有的 c++ 也有！（让我有种扬眉吐气的感觉）并且在 c++98 就支持了。 奇怪的是他没有迭代器，这让我百思不得其解，既然他也是从 std::vector 继承来的， 为什么不定义一个迭代器呢？只能通过下标访问，唯一的解释就是它要和数学运算挂钩。 （但这也不影响它加一个迭代器啊？）
